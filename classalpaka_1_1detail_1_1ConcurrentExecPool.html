template<typename TConcurrentExecutor, template< typename TFuncReturn > class TPromise, typename TCurrentException, typename TYield, typename TMutex = void, template< typename TMutex2 > class TUniqueLock = std::atomic, typename TCondVar = void, bool TbYield = true>
class alpaka::detail::ConcurrentExecPool< TConcurrentExecutor, TPromise, TCurrentException, TYield, TMutex, TUniqueLock, TCondVar, TbYield >

ConcurrentExecPool using yield.

Template Parameters
TConcurrentExecutorThe type of concurrent executor (for example std::thread).
TPromiseThe promise type returned by the task.
TCurrentExceptionMust have a static method "current_exception()" that returns the current exception.
TYieldThe type is required to have a static method "void yield()" to yield the current thread if there is no work.
TMutexUnused. The mutex type used for locking threads.
TUniqueLockUnused. The lock type used to lock the TMutex.
TCondVarUnused. The condition variable type used to make the threads wait if there is no work. Uses the TUniqueLock.
TbYieldBooleam value the threads should yield instead of wait for a condition variable.

Constructor.

Creates a concurrent executor pool with a specific number of concurrent executors and a maximum number of queued tasks.

Parameters
uiConcurrentExecutionCountThe guaranteed number of concurrent executors used in the pool. This is also the maximum number of tasks worked on concurrently.
uiQueueSizeThe maximum number of tasks that can be queued for completion. Currently running tasks do not belong to the queue anymore.
