Detailed Description

The fibers accelerator.

This accelerator allows parallel kernel execution on the host. It uses boost::fibers to implement the cooperative parallelism. By using fibers the shared memory can reside in the closest memory/cache available. Furthermore there is no false sharing between neighboring threads as it is the case in real multi-threading.
